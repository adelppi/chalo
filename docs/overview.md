# Overview — chalo

## 一行で

> カップルが「いつか一緒に行きたい所・やりたい事」をサクッと貯めて、共有できるようにするアプリ。

## ビジョン

ふたりの体験を高める。

## ターゲット

- 一緒に出かける相手がいるカップル（恋人・夫婦・パートナー）。
- 「行きたいところリスト」をメモアプリやチャットでなんとなく管理していて、流れて消えてしまっている人。
- iOS ユーザー、日本語話者。日本国内・日本語のみ。

## 提供価値

- 一緒にやりたいことを思いついたら、詳細が決まっていなくても、忘れる前にすぐ相手と共有できる。
- 期限のあるもの（特別展・期間限定メニューなど）は、メモやチャットのやり取りと違い、やり損ねる前に通知で知らせる。
- 簡単な操作で、使いやすく、高機能にしすぎず提供する。chaloが負担にならない。

## コアループ

思いついたらタイトルだけで貯める。日付を入れると予定になり、その日を過ぎると自動で「おしまい」になる。ステータスの定義と遷移は `domain/plan-lifecycle.md` を正とする。

## やらないこと（プロダクトの考え方として持たないもの）

判断のブレを防ぐため、明示的に外した選択がある。これらは時間の問題ではなく、**chalo の思想として持たない**と決めたもの。

考え方の軸は「ユーザーに登録・管理させない」「プランは全共有でサプライズを作らない」「アルバムは端末ローカルで完結し共有しない」「操作と通知を増やさない」。

持たないものの**完全な一覧と理由は `features.md`「chalo が持たないもの」を正とする**。

## 前提

土台となる技術・体裁の前提を固定する。

- プラットフォーム：iOS のみ
- 言語：日本語のみ
- 収益：無料を前提に設計（プレミアム余地はメモのみ。`features.md` 参照）
- 開発：React Native + Expo、AI駆動開発、EAS 積極活用（特に `eas update`）
- バックエンド：Supabase（認証・DB・同期）
- 外観：ライトモードのみ（ダークモード非対応）
- マスコット：チャロくん（犬×たぬき。足跡・しっぽ・顔）

## 成功の見方（参考・要検討） [提案]

数値目標は厳密に置かないが、設計判断の拠り所として「定着」を最重視する。具体的には、プランを貯め続けてくれるか／おしまいになったプランのアルバムを開いてくれるか／振り返り通知から再来訪してくれるか、の3点をKPI候補とする。
