# 機能一覧（Features）— chalo

完成した chalo が持つ機能の一覧。各機能の詳細仕様は `domain/` および `non-functional.md` を参照。
仕様の細部（値・形式・方式）で未決のものだけ `open-questions.md` に集約する（機能の有無ではなく、詳細を詰めるための一覧）。

末尾に「**chalo が持たないもの（設計判断）**」を置く。これは未実装ではなく、プロダクトの考え方として持たないと決めたもの。

## chalo が持たないもの（設計判断）

プロダクトの考え方として持たないと決めたもの。判断のブレを防ぐために明記する。

| 持たないもの | 理由 |
|---|---|
| 写真の手動登録 | アルバムは「その日の写真を自動で拾う」だけ |
| 共有アルバム（クラウド共有） | 会って一緒に開けば見せ合える。アップロードせず端末ローカルで完結 |
| アルバムの場所（ジオタグ）絞り込み | その日に撮ったものは全部その日の思い出。抽出は日付のみ |
| メモ以外の自由記述（下書き・個人メモ） | 持たない |
| サプライズ／相手に隠すプラン | プランは全共有 |
| チェックリスト／サブタスク | TODO感は「プランをおしまいにしていく」ことで担保 |
| カテゴリ／タグ | ユーザー負荷を下げ、機能を洗練する |
| 相手とのカレンダー同期 | カレンダーは各自の端末への一方向追加のみ |
| アプリ内カレンダー画面 | プランは一覧で管理する |
| おしまいの取り消し（再オープン） | 操作を増やさない。誤って閉じたら再作成で対応 |
| 通知タイミングのユーザーカスタム | 厳選した既定で送る |
| 通知の煽り（新しい予定を促す） | 既にあるものを知らせるだけで、新しい予定は促さない |

## 収益化

- 無料。
- プレミアムの余地（例：書き出し拡張など）はメモのみで、設計を縛らない（`open-questions.md` Q-BIZ-1）。
