# 用語集（Glossary）— chalo

人とAIが同じ言葉で話すための辞書。実装・UI文言・ドキュメントはこの語彙に揃える。英語表記はコード上の識別子の目安。

## コア概念

### プラン（Plan）
chalo の中心となるデータ。「行きたい所・やりたい事」一つ分。**A案＝1プラン＝1TODOアイテム**。詳細・日付・場所が無くても、タイトルだけで成立する。後から編集できる。
- 持つフィールド：タイトル（必須）、日付、時刻、期限、場所、参考URL、メモ、ステータス、作成者。詳細は `data-model.md`。

### いつかやることリスト / someday list
日付も予定も決まっていないプランを含む、貯めたプラン全体の心象。chalo の核となる体験。UI上は「やりたい」と表現する。

### ステータス（Status）
プランのライフサイクル上の状態。`やりたい` / `予定` / `おしまい`。詳細は `domain/plan-lifecycle.md`。

### やりたい（未定 / Wish）
日付が入っていないプラン。いつ行くか未定のまま、いくつでも貯めておける状態。

### 予定（Scheduled）
日付が入ったプラン。行く日が決まった状態。

### おしまい（Done / Closed）
体験が完了した状態。chalo らしい温かい呼び名としてこの語を正式採用する（"完了" "達成" ではなく "おしまい"）。
- **自動おしまい**：日付ありプランが、予定日の終わり（時刻があればその時刻以降）に自動で閉じる。
- **手動おしまい**：「おしまい」ボタンを押して閉じる。日付なしプランもこれで閉じられる。

### おしまい日（Closed date）
プランが「おしまい」になった日。自動なら予定日（保存せず導出）、手動なら押した日（記録）。**振り返り通知**と、日付なしプランの**アルバム対象日**の基準になる。

### 思い出基準日 / アルバム対象日（Memory anchor date）
アルバムが写真を拾う対象の日。**予定日があればその日、なければおしまい日**。標準フロー（日付ありは予定日で自動おしまい）では予定日＝おしまい日となり一致する。

### アルバム（Album）
おしまいになったプランに紐づく、その日に撮られた写真の自動コレクション。**端末ローカルの写真のみ**を、撮影日付（スクショ含む）で抽出して表示する。アップロードしない。共有しない。詳細は `domain/album.md`。

### 期限（Deadline）
「これを過ぎると行けなくなる」日。特別展の終了日、期間限定メニューの終了日など。**行く予定日（日付）とは別物**。期限を入れること自体が「終わる前に通知してほしい」という意思表示になる。詳細は `domain/notifications.md`。

## ペア関連

### ペア（Pair）
1対1で結ばれた二人の関係。chalo は常に二人一組での利用を想定する。

### パートナー（Partner）
ペアの相手。

### 相手の呼び方（Partner nickname）
ユーザーが設定でパートナーをどう呼ぶか（表示名）。

### 招待コード（Invite code）
ペアを成立させるためのコード。一方が発行し、もう一方が入力して繋がる。手入力方式で、**6桁の数字**・有効期限は発行から**24時間**（`domain/pairing.md`）。

### ソロ利用（Solo）
ペアを招待するまでの一人の状態。プランを貯めておける、試用・事前登録の期間。chalo は一人での継続利用は想定しない。

### ロック状態（Locked）
パートナーが退会したあと、残った側が入る状態。できるのは「全プランの書き出し」か「アカウント削除」だけ。通常利用・新規招待はできない。詳細は `domain/pairing.md`。

### 書き出し / エクスポート（Export）
全プランの内容を連結した1つのテキスト（.txt）を生成し、iOS共有シートで共有する常設バックアップ機能。アルバム写真は対象外。

## 同期・編集

### 同期（Sync）
ペア間でプランのデータを揃えること。タイミング・方式は `adr/0004-sync-strategy.md` を参照。

### 編集ロック（Edit lock）
相手が編集中のプランを同時に編集しないための仕組み。編集を始めるとロックがかかり、相手には「〇〇が編集中です」と表示される。**TTL（一定時間で自動失効）あり**。詳細は `adr/0005-edit-lock.md`。

### 競合検知（Conflict detection）
編集ロックによって、二人が同じプランを同時に編集する状況を防ぐこと。

## 機能・UI

### 参考URL（Reference URL）
プランに1つだけ持てる、タップで遷移できるURL文字列。ドメインに応じて左にアイコンを出し分ける（Instagram / X / 食べログ / その他は汎用アイコン）。

### カレンダー連携（Calendar）
プランを各自の端末カレンダーへ一方向で追加する機能。相手とは同期しない。追加済みのプランを編集するとイベントも自動更新、削除すると自動削除される。詳細は `domain/calendar.md`。

### 触覚フィードバック（Haptics）
意味の大きい4操作（プラン作成・手動おしまい・ペア成立・プラン削除）に与える振動フィードバック。全操作には付けない。

### 空状態（Empty state）
プランが0件のときなどの表示。チャロくんのグレーの足跡を画面中央に置く。

## マスコット・ブランド

### チャロくん（Chalo）
chalo のマスコット。犬とたぬきのニュアンスを併せ持つ。素材は顔・しっぽ・足跡。Rive で「揺れ・瞬き程度」の控えめなアニメーションを付ける。

### 足跡（Pawprint）
チャロくんのモチーフ。ローディング・スプラッシュ・空状態などに用いるブランド要素。「行った跡／旅の証」を表す。

## 通知

### 作成通知（Creation push）
パートナーがプランを作成したときに飛ぶプッシュ通知。頻度・条件は `domain/notifications.md`。

### 期限通知（Deadline push）
期限のあるプランを、終わる前に知らせる通知。タイミング・送信条件は `domain/notifications.md`。

### 振り返り通知（Reminiscence push）
1年前におしまいになったプランについて、また行ける頃に知らせる通知。詳細は `domain/notifications.md`。
