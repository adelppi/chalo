# ドメイン: 端末カレンダー連携

プランを各自の端末カレンダーへ**一方向で追加**する。相手とは同期しない。

この文書は連携の**振る舞い**を定義する。技術的な実現方法（ライブラリ・リンクの保存先など）は `adr/0012-calendar-integration.md` を正とする。

## 思想 [確定]

- chalo 内でプランを作り、プラン詳細から端末カレンダーに反映する。
- **相手とのカレンダー同期はしない**（各自の端末ローカルの操作）。
- アプリ内カレンダー画面は持たない（プランは一覧で管理する）。

## ボタンと状態 [確定]

ボタンは連携状態に依存する。

| 状態 | ボタン | 動作 |
|---|---|---|
| 未連携 | 「カレンダーに追加」 | 端末カレンダーにイベントを作成し、リンクを保存 |
| 連携済 | 「カレンダーから削除」 | 端末カレンダーのイベントを削除し、リンクを解除 |

> 表示は「追加 / 削除」とする。「同期」は双方向（カレンダー→プラン）に聞こえるため避ける。裏側の自動更新が "同期" の役を果たす。

- **追加・削除の成功時はトーストで知らせる**（「カレンダーに追加しました」「カレンダーから削除しました」）。
- **「カレンダーから削除」は確認ダイアログなしで即実行**する（端末カレンダー側だけの操作で、プラン本体は消えないため）。
- **日付なしプランは追加不可**（ボタン非活性。日付を入れると追加できる旨の補足を添える）。日時が無いとカレンダーに載せられないため。

## カレンダーイベントに渡す内容 [確定]

- タイトル
- 日時（日付＋時刻。時刻が無ければ終日イベント）
  - **[提案]** 時刻ありイベントの長さは開始から**1時間**とする（iOS カレンダーの既定に合わせる）。
- 場所
- メモ
- **メモ末尾に目印**：「チャロが追加🐾」を入れる。リンクが失われた時の判定にも使う。
- **追加先プロファイル**：どのカレンダー（プロファイル）に入れるかを使う。既定は**設定画面で選んだ既定カレンダー**を用いる。**未選択時は端末のデフォルトカレンダーにフォールバック**する。

## リンクの保持 [確定]

- カレンダー操作は各自の端末ローカルなので、**プランと端末イベントの対応もローカルに保持**する。
- これにより「このプランは端末カレンダーに追加済みか」「どのイベントか」を判定できる。
- 対応の保存形式は `adr/0012-calendar-integration.md` を参照。

## 置いてけぼり防止（自動更新・自動削除） [確定]

押し直さなくても古い予定が残らないようにする。

- **連携済みプランの編集 → イベント自動更新**：日付・時刻・タイトル・場所・メモのいずれかが変わったら、対応する端末イベントを更新する。
- **連携済みプランから日付を外した → イベント削除＋リンク解除**：日付なしではカレンダーに載せられないため、イベントを消して「未連携」に戻す。
- **プラン削除 → イベント自動削除**：プランを消したら、対応する端末イベントも削除する。

## 例外・エラー [確定]

- **外部で手動削除された**（ユーザーが端末カレンダーアプリでイベントを消した）：更新・削除しようとして失敗を検知したら、**リンクを解除して「未追加」状態に戻す**。
- **追加 / 更新 / 削除に失敗**：トースト＋リトライ（`non-functional.md`）。
- カレンダー権限が未許可：機能を縮退し、設定のカレンダー許可ボタン → iOS 設定への導線。

## 権限・設定 [確定]

- カレンダーアクセス許可が必要。許可の取得・拒否時の縮退方針は `non-functional.md`（権限）を正とする。
- 設定画面に、カレンダー許可のオンボタンと、**既定の追加先カレンダー選択**を置く。

## 持たないもの

カレンダー連携が持たないもの（相手との同期／アプリ内カレンダー表示）は `features.md`「chalo が持たないもの」を正とする。
