# ADR-0010: マスコット「チャロくん」とアニメーション（Rive）

- ステータス: 採用 [確定]
- 関連: adr/0006

## コンテキスト

ブランドの核としてマスコット「チャロくん」（犬×たぬき。素材は顔・しっぽ・足跡）を据える。chalo（=行こう）と足跡（=旅の証）がコンセプトと噛み合う。アニメーションは欲しいが、派手にはしない。

## 決定

- アニメーションは **Rive** で実装する。
- 演出は **「揺れ・瞬き程度」** に統一する（デザイン原則）。控えめに動く範囲を上限とし、派手な動きはしない。
- 出番：
  - スプラッシュ／ローディング：しっぽ・足跡を緩やかに表示。実装は次項の例外により GIF。
  - 空状態（0件など）：グレーの足跡を画面中央に。
  - おしまい時の小さな演出：足跡スタンプ [提案]。
- 技術前提：`rive-react-native` は **Expo Go では動かず、Dev Client / EAS Build が必要**（`adr/0006` の方針と整合）。

### 例外：起動直後のローディング画面は GIF（Issue #36）[確定]

- ネイティブの起動スプラッシュ（`expo-splash-screen`、`app.json` の `expo.plugins`）は静止画のみでアニメーション不可。JSマウント後に表示する引き継ぎ用のローディング画面（`SplashLoading`）は、既存アセット `splash_chalo_paw_print.gif` をそのまま流用する。
- Rive への統一はこの箇所には適用しない。将来 Rive に置き換える場合は本 ADR を更新する。

## 結果

- 良い点：軽量で表現力のあるアニメをブランドに沿って出せる／世界観の一貫性。
- 留意点：ネイティブ依存のため OTA では Rive ランタイム自体は更新不可。Rive ファイル（アセット）の差し替えは扱い方を確認のうえ運用する。

## デザイン原則（明文化）

- 動きは控えめ。ユーザーの操作を妨げない。
- ライトモードのみ（`non-functional.md`）。
- チャロくんは急かさず寄り添うトーン。通知の思想（煽らず、終わる前に知らせる）と一致させる。
