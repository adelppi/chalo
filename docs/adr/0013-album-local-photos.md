# ADR-0013: アルバムは端末ローカル写真を日付で都度取得する

- ステータス: 採用 [確定]
- 関連: domain/album.md, adr/0001

## コンテキスト

おしまいになったプランに「その日の写真」を自動で見せる。抽出ルール・思想・空状態などの振る舞いは `domain/album.md` で定義する。本 ADR はその技術的な実現方法を定める。

## 決定

- 写真取得は **`expo-media-library`**（または同等）で、撮影日（creationDate）でフィルタする。
- アルバムの写真は **アプリ側で保存・キャッシュしない**。表示のたびに対象日でライブラリから引く。表示が重い場合のみサムネイルの短期キャッシュを検討する。
- 写真は **端末外に出さない**（Supabase Storage にも置かない。`data-model.md`）。

## 結果

- 良い点：アップロード不要でプライバシーが保てる。実装が単純。
- 留意点：限定アクセス（Limited Photo Access）では対象日の写真が見えないことがある。空表示＋アクセス拡大の導線で対応する（振る舞いは `domain/album.md`、`non-functional.md`）。
- 性能：対象日のアセットだけを日付クエリで取得し、サムネイルを遅延ロードする。全件走査は避ける（`non-functional.md`）。

## 検討した代替案

- **撮影日に加えジオタグで絞る**：場所一致の写真だけを集める案。chalo の思想（その日に撮ったものは全部その日の思い出）に反するため採らない。
