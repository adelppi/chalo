# ADR-0002: ナビゲーションに Expo Router を採用する

- ステータス: 採用 [確定]

## コンテキスト

RN + Expo 構成。タブ ＋ 詳細/編集/設定のサブ画面。通知タップからの**ディープリンク**（該当プラン詳細へ遷移）が要件。

## 決定

**Expo Router**（ファイルベースルーティング）を採用する。

- タブ・スタックをディレクトリ構成で表現。
- 通知タップ → 該当プラン詳細へのディープリンクを URL ルートで扱う。
- AI駆動開発との相性：画面＝ファイルの対応が明確で、AIに画面追加を指示しやすい。

## 結果

- 良い点：Expo標準で統合が楽／ディープリンクが素直／構成が規約化されAIに優しい。
- 留意点：細かな遷移制御は React Navigation の知識が前提（Expo Router は内部でこれを使う）。

## 検討した代替案

- **React Navigation 直接利用**：自由度は高いがボイラープレートが増える。Expo Router がこれをラップするため、まず Expo Router を選ぶ。
