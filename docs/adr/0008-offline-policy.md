# ADR-0008: オフライン方針（オフライン時はブロック）

- ステータス: 採用 [確定]
- 関連: adr/0003, adr/0004, non-functional.md

## コンテキスト

chalo はオンラインで使うことを前提とする。オフラインでの作成・編集を許すと同期キュー・競合解決が必要で複雑になる。読み取り専用で古いキャッシュを見せる案も、「いつのデータか」の曖昧さを生む。シンプルさを優先し、オフライン時は一貫してブロックする。

## 決定

- オフライン時は**アプリを使えない状態にする**（読み取り専用でも見せない）。
  - アプリを開いた（または操作中に接続が切れた）ら、全画面の**「オフラインです」表示**に切り替える。
  - インターネットの復旧を促し、**復旧してから戻ってくる**よう案内する。復旧を検知（または再試行）したら通常画面へ戻す。
- キャッシュの有無で挙動を分けない。オフラインなら常にこのブロック表示にする。
- OIDCログインもネット必須のため、未ログイン時も同じくオフラインでは進めない。

## 結果

- 良い点：同期キュー・オフライン競合の複雑性を持ち込まない。「いつのデータか」の曖昧さもなく、実装と検証が軽い。
- 留意点：オフラインでは一切閲覧・操作できない。利用シーン（多くはオンライン）から許容と判断。
- オフライン作成／閲覧が必要になれば別ADRで検討する。

## 検討した代替案

- **読み取り専用でキャッシュを表示**：オフラインでも直近を見られるが、古いデータの見せ方・同期境界の説明が要る。一貫してブロックする方が単純。採用しない。
- **オフライン作成をキューに積む**：体験は良いが、同期キュー・順序・競合解決が必要で重い。採用しない。
