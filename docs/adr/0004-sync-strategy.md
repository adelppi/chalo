# ADR-0004: 同期は起動/復帰時の再取得とする

- ステータス: 採用 [確定]
- 関連: adr/0003, adr/0005, adr/0008

## コンテキスト

ペア間でプランを揃える必要がある。常時リアルタイム同期は実装・接続コストが高い。chalo の利用は「思いついた時に貯める」「たまに見る」性質で、秒単位の即時反映は必須でない。

## 決定

- 同期は **アプリ起動時・フォアグラウンド復帰時に再取得**する。Realtime 常時購読は使わない。
- 編集ロックの判定だけは別扱い：**編集ボタンを押した瞬間に、その1件だけ最新取得**してロック状態を確認する（`adr/0005`）。

## 結果

- 良い点：実装がシンプル／接続コストが低い／TanStack Query と素直に噛み合う。
- 留意点：相手の変更がリアルタイムには映らない（次回の起動/復帰で反映）。編集ロックは「押した瞬間取得」で担保するためリアルタイム不要。
- リアルタイム反映が必要になれば Supabase Realtime を後付けできる（別ADRを追記）。

## 検討した代替案

- **Supabase Realtime 常時購読**：即時反映できるが、chalo の体験には過剰。バッテリ・接続管理のコストも増える。
