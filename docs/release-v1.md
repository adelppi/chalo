# 初回リリース（v1）スコープ — chalo

初回リリース（v1）で実装する範囲を決める文書。

## 位置づけ [確定]

- `features.md` は完成した chalo の姿の正であり、本文書はそれを変えない。「持たないもの」の一覧にも影響しない。
- 本文書が決めるのは**実装の順序**だけ。「v1では見送る」とした機能は、完成形には含まれる。
- v1 リリース後は、見送り機能の実装状況をここで追う。

## 方針 [確定]

- **配布：App Store 一般公開**。TestFlight 限定にはしない。したがって公開要件（`non-functional.md`「App Store 公開要件」）は v1 のスコープに含まれる。
- **判断基準：コアループとペア共有を完成させることを最優先**。貯める→予定→おしまいの流れと、ふたりでの共有が v1 の本体。
- プランが貯まって初めて効く機能（検索・並び替え、振り返り通知、アルバム）は見送る。これらの多くは OTA（`adr/0006`）で後追いできる。ただし**ネイティブモジュールを伴うものは再ビルド＋審査提出が必要**（各項の注意に明記）。

## v1 では見送るもの [確定]

完成形（`features.md`）には含まれる。後続リリースで実装する。

| 機能 | 見送る理由 | 追加するときの注意 |
|---|---|---|
| 3.1 自動アルバム | コアループに直結せず、リリース直後は「おしまい」がまだ少ない | 写真ライブラリはネイティブモジュールのため**再ビルド＋審査提出が必要**。写真権限の文言・限定アクセス挙動・プライバシーラベルの写真項目もこのとき追加（`adr/0013`） |
| 5.3 振り返り通知 | 対象は「1年前のおしまい」であり、リリースから1年間は発火しない | サーバ定期バッチが初めて必要になる。Q-NOTIF-2 もこのとき確定 |
| 2.3 あいまい検索 / 2.4 並び替え・絞り込み | プランが貯まってから効く機能 | OTA で後追い可。Q-FEAT-1 をこのとき確定 |
| 10.2 マスコットの控えめアニメ | 静的表示で体裁は成立する | OTA で後追い可（実装方式は `adr/0010`。ネイティブ依存が出る場合は再ビルド） |
| 11.6 クラッシュ/エラー監視 | v1 は端末内ログ＋不具合報告（9.8・11.4）で拾う | 導入判断は Q-OPS-1。入れる場合はプライバシー配慮 |

## App Store 公開要件の扱い [確定]

`non-functional.md`「App Store 公開要件」を v1 で満たす。ただし**写真ライブラリ関連（利用目的文言・限定アクセス挙動・プライバシーラベルの写真項目）は自動アルバム見送りに伴い v1 では対象外**。アルバム追加時に併せて対応する。
