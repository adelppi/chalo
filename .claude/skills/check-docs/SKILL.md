`docs/` 全体の整合性を検査し、結果を報告する。**修正はユーザーの承認を得てから**行う。

## 検査項目

1. **リンク・参照**: docs 内の相対リンクや `adr/XXXX`・`domain/xxx.md` 形式の参照が、実在するファイルを指しているか。
2. **目次との一致**: `docs/README.md` の構成ツリーと `docs/` の実ファイルが一致するか。片方にしかないファイルはないか。
3. **一次情報の重複**: `docs/README.md`「一次情報は1か所だけ」の表に載っている事実が、正とされる文書以外で再定義(リンクではなく再掲)されていないか。
4. **ステータス印の同期**: `[保留]` の付いた記述が `docs/open-questions.md` に登録されているか。逆に、解決済みの論点が `[保留]` のまま本文に残っていないか。
5. **ADR の健全性**: 採番が連続しているか、各 ADR の `関連:` の参照先が実在するか、廃止(superseded)された ADR を正として参照している箇所がないか。
6. **コードとの乖離**(`src/` が存在する場合のみ):
   - adr/0015 の依存ルール違反(feature 間の直接 import、`global/lib/supabase` 以外での supabase-js の import)がないか。
   - `src/features/` の一覧と adr/0015 の feature 割り付け・`docs/features.md` が整合しているか。

## 報告形式

- 問題を「**重要**(事実の矛盾・リンク切れ・ルール違反)」と「**軽微**(表記ゆれ・目次の抜け)」に分けて列挙する。
- それぞれ該当ファイルと行、修正案を添える。
- 問題がなければ「問題なし」と明確に報告する。検査した項目の一覧も添える。
